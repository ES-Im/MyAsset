# 도메인 모델

### 도메인간 관계 정의
- USER (1) --- (0..1) USER_INFO
## 사용자(user)

---
### _Entity : user_
 
- 내부관리용 회원관련 비개인정보 엔티티 
####  Attribute

| 속성            | 설명         |
|---------------|------------|
| userid        | 사용자의 내부식별자 |
| status        | 사용자 상태     | 
| role          | 관리자여부      | 
| creatAt       | 가입일자       | 
| lastLoginAt   | 마지막 로그인 일자 | 
| withdrawReqAt | 탈퇴요청일      |  

    ※ Enum : status
      · ACTIVE : 활성계정
      · WITHDRAWN : 탈퇴
      · WITHDRAW_PENDING : 탈퇴요청
      · DORMANT : 휴먼계정
    
    ※ Enum : role
      · ADMIN : 관리자
      · USER : 일반 사용자

### _Entity : user_info_

- OAuth로 불러온 회원관련 개인정보 엔티티
#### Attribute
| 속성 | 설명 |
|----|----|
|  userId  |  사용자의 내부식별자  |
|  name  |  사용자 이름 (소셜서버에서 받은 값)  |
|  email  |  사용자 이메일 (소셜서버에서 받은 값)  |
|  providerType  | OAuth 서비스 Provider   |

    ※ Enum : providerType
      · KAKAO
      · GOOGLE
      · NAVER

### 행위
    - constructor(): 회원등록 (status=ACTIVE, role=USER)
    - requestWithdraw(): 회원탈퇴요청 
    - login(): 소셜로그인

### 정책 (Batch / Scheduler)
    - markDormant: 휴먼상태변경 
    - markWithdrawn: 회원탈퇴 상태 전환 
    - deleteUserPermanently: 회원정보 물리적으로 삭제 

### 규칙
    ① status 변환시점
      · ACTIVE : 소셜 회원가입 즉시
      · ACTIVE → DORMANT : 1년간 미접속
      · ACTIVE → WITHDRAW_PENDING : 사용자 탈퇴요청
      · WITHDRAW_PENDING → WITHDRAWN : 사용자 탈퇴요청 후 30일 이후 90일 이내
      · WITHDRAW_PENDING / DORMANT → ACTIVE : 소셜로그인 && 사용자 동의

    ③ 회원탈퇴 
      · 사용자는 ACTIVE상태에서 회원을 탈퇴(요청)할 수 있다.
      · WITHDRAW_PENDING 상태에서 30일이 지나면 해당 유저의 개인정보(E.user_info)가 삭제된다. 
      · WITHDRAWN 상태전환 후 60일이 지나면 해당 유저에 대한 정보(E.user) 전부 물리적으로 삭제된다.
